Ver detalles de mi empresa
GET/fabricante/

Retorna los detalles de mi perfil empresarial de fabricante en abi.

Campos de cada objeto:

campo tipo descripcion
id integer id de fabricante
nombre string nombre de fabricante
imagen_perfil integer url de imagen de perfil de fabricante
aprobado boolean está o no aprobado por el staff
fecha_agregado string fecha en formato ISO en la cual se registró
aprobado_por string email del usuario quien aprobó al fabricante
puntos_retiro object/null puntos de retiro disponibles
usuarios object listado de usuarios relacionados al fabricante
rubros object rubros del fabricante
ruc string ruc del fabricante
razon_social string razón social del fabricante
nro_telefono string número de teléfono del fabricante
resena_empresa string/null reseña del fabricante
api_enabled boolean está o no habilitada la api
ip_safelist string listado de IP’s habilitadas para el acceso la api
rating float calificación del fabricante
ratings integer calificación del fabricante
comision float comisión del fabricante
tipo_proveedor integer indica el tipo de fabricante (10 -> emprendedor, 20 -> empresa)
fecha_aceptacion_contrato string fecha en formato ISO en la cual se aceptó el fecha_aceptacion_contrato
constancia_ruc string url de la imagen de la constancia de ruc
bloqueado boolean está o no bloqueado
dias_apertura object dias de apertura del negocio (1 -> lunes, 2 -> martes, 3 -> miércoles, 4 -> jueves, 5 -> viernes, 6 -> sábado, 7 -> domingo )
hora_apertura string hora de apertura del negocio
hora_cierre string hora de cierre del negocio
celular string número de celular del fabricante
email string correo electrónico del fabricante
sitio_web string url del sitio web del fabricante
horario_atencion string descripción del horario de atención del fabricante

Campos del punto de retiro dentro del perfil empresarial

campo tipo descripcion
id integer id del punto de retiro
fabricante integer id del fabricante
nombre string nombre del punto de retiro
calle_principal string nombre de la calle principal del punto de retiro
calle_secundaria string nombre de la calle secundaria del punto de retiro
numero string número de casa del punto de retiro
referencia string referencia para llegar al punto de retiro
ubicacion string latitud y longitud donde se encuentra el punto de retiro
default boolean es punto de retiro por defecto
ciudad integer id de la ciudad relacionada
nro_telefono string/null número de teléfono del fabricante

Campos de usuarios dentro del perfil empresarial

campo tipo descripcion
id integer id del usuario
username string nombre de usuario
first_name string nombre del usuario
last_name string apellido del usuario
nro_telefono string número de teléfono del usuario
email string correo electrónico del usuario
nro_documento string número de documento del usuario
ruc string ruc del usuario
razon_social string razón social del usuario
imagen_perfil string url de la imagen de perfil del usuario
direcciones object listado de direcciones registradas
fecha_nacimiento string fecha de nacimiento en formato ISO
genero integer genero del usuario (10 -> femenino, 20 -> masculino, 30 -> otro)
email_oferta boolean está suscrito para recibir ofertas por email
notificacion_oferta boolean está suscrito para recibir ofertas por notificaciones
datos_facturacion object datos de facturación del usuario

Campos de direcciones dentro de usuarios

campo tipo descripcion
id integer id de la dirección

default boolean es dirección por defecto

Crear producto
POST/producto/

Crea un producto nuevo. El producto queda en estado pendiente, el staff debe moderarlo para que aparezca en la plataforma.

Campos de cada objeto:

campo tipo requerido por defecto descripcion
id integer si autonumerico id del producto
fabricante integer si - id del fabricante
categoria integer si - id de categoria de producto
nombre string si - nombre del producto
descripcion string si - descripcion corta del producto
descripcion_extensa string si - descripcion extensa del producto, incluye texto estilizado
etiquetas string no “” etiquetas de producto, separadas por coma, ejemplo: “tecnologia, ropas”
cantidad_mayorista integer si 0 cantidad de productos a comprar para activar el precio mayorista
precio integer si 0 precio normal de venta de producto
precio_mayorista integer si 0 precio mejorado de producto
cantidad_disponible integer si 0 stock disponible para la venta
peso_kg float no 0 peso del producto en kilogramos
alto_cm float no 0 alto del producto en centimetros
ancho_cm float no 0 ancho del producto en centimetros
profundidad_cm float no 0 profundidad del producto en centimetros
refrigeracion integer no 0 refrigeracion requerida para el producto (0 -> no refrigerado, 10 -> frio, 20 -> congelado)
api_extra_data string no “” campo extra para los fabricantes. no se usa en el sistema.
video string no null video del producto
especificaciones string no “-” especificaciones del producto
fecha_reposicion string no null fecha en formato ISO de la reposición del producto
cantidad_reposicion integer no null cantidad a reponer
link string no “” link del producto en la tienda ABi

Campos de imagenes dentro de producto:

campo tipo descripcion
id integer id de la imagen
imagen string url de la imagen
producto integer id del producto
portada boolean es o no la imagen principal
resoluciones object tamaños disponibles de la imagen

Notificar orden nueva
POST/endpoint_definido_por_fabricante

El sistema realiza un POST a la url definida por el fabricante, a modo de recibir las nuevas ordenes que vayan llegando.

En el header del request, se envia el token de acceso de fabricante, se recomienda verificar, para proteger contra intentos de vulnerar el sistema.

El request contiene la orden de venta nueva.

Para los campos del objeto, ver Ver orden de venta
